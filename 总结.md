# PowerShell 配置修复总结

## 问题
- `Microsoft.PowerShell_profile.ps1` 中 oh-my-posh 初始化仅依赖 `$env:POSH_THEMES_PATH`，在该变量未设或主题不在该路径时无法加载指定主题。

## 原因
- 假设 oh-my-posh 及主题安装路径固定，缺乏对常见安装目录和 CLI 缺失的判断，也未对相关模块缺失进行容错处理。

## 修复方法
- 导入 `posh-git` 与 `PSReadLine` 时添加 `-ErrorAction SilentlyContinue`，避免缺失报错中断。
- 检查 `oh-my-posh` 是否存在，按顺序在 `$env:POSH_THEMES_PATH` 与 `%LOCALAPPDATA%\Programs\oh-my-posh\themes` 中定位 `robbyrussell.omp.json`，无匹配时回退至默认配置。
- 若未检测到 CLI，提示使用 `winget install JanDeDobbeleer.OhMyPosh` 安装，确保不同安装方式均可使用。
